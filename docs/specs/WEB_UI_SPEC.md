# Web UI ダッシュボード - 仕様書

> Discord Bot「Ayasono」の管理用 Web ダッシュボード。各サーバーの管理者が Bot の設定をブラウザから閲覧・編集できる。

最終更新: 2026年3月29日

---

## 概要

### 目的

Discord のスラッシュコマンドで行えるすべての設定操作を、Web UI からも実行可能にする。コマンドより直感的でミスの少ない設定体験を提供する。

### 機能一覧

| 機能 | 概要 |
| --- | --- |
| 認証 | Discord OAuth2 によるログイン・ログアウト |
| サーバー選択 | 管理権限を持つサーバー一覧から選択 |
| ダッシュボード | サーバーの概要・機能有効状態の一覧 |
| サーバー設定 | 言語・エラーチャンネルの設定・インポート/エクスポート |
| AFK 設定 | 有効/無効・AFK チャンネルの設定 |
| Bump リマインダー設定 | 検知チャンネル・メンション対象の管理 |
| VAC 設定 | 有効/無効・トリガーチャンネルの管理 |
| メンバーログ設定 | ログチャンネル・メッセージテンプレートの設定 |
| VC 募集設定 | セットアップ・メンションロールの管理 |
| チケット設定 | パネルの作成・編集・削除、各種オプション |
| リアクションロール設定 | パネルの作成・編集・削除、ボタン管理 |
| メッセージ固定設定 | メッセージの作成・編集・削除 |

### 対象ユーザー

| ユーザー種別 | アクセス条件 | 操作範囲 |
| --- | --- | --- |
| サーバーオーナー | Discord OAuth2 + サーバーオーナー | 全設定の閲覧・編集 |
| サーバー管理者 | Discord OAuth2 + ManageGuild 権限 | 全設定の閲覧・編集 |
| 一般ユーザー | - | アクセス不可 |

### Bot 権限モデル

Web UI からの Discord API 操作は Bot トークンを使用するため、Bot に必要な権限が付与されている必要がある。**管理者（Administrator）権限の付与を推奨。**

**機能別の必要権限:**

| 機能 | 必要な Bot 権限 |
| --- | --- |
| AFK | MoveMembers |
| Bump リマインダー | SendMessages, EmbedLinks |
| VAC | ManageChannels, MoveMembers, SendMessages, EmbedLinks |
| メンバーログ | ViewChannel, SendMessages, EmbedLinks, ManageGuild |
| VC 募集 | SendMessages, ManageMessages, EmbedLinks, CreatePublicThreads, ManageThreads, ManageChannels |
| チケット | ManageChannels, ManageRoles, SendMessages |
| リアクションロール | ManageRoles, SendMessages |
| メッセージ固定 | SendMessages, ManageMessages, EmbedLinks |
| サーバー設定 | AttachFiles |

> Bot に必要な権限が不足している場合、該当操作の API は `DISCORD_API_ERROR`（502）を返し、フロントエンドで不足権限を表示する。

## 認証・認可

### Discord OAuth2 フロー

```
ユーザー → [ログインボタン] → Discord 認可画面 → コールバック → セッション発行 → サーバー選択画面
```

**OAuth2 スコープ:**

| スコープ | 用途 |
| --- | --- |
| `identify` | ユーザー情報（ID、ユーザー名、アバター）の取得 |
| `guilds` | ユーザーが所属するギルド一覧の取得 |

### セッション管理

| 項目 | 仕様 |
| --- | --- |
| 方式 | サーバーサイドセッション（Cookie ベース、DB ストア） |
| セッションストア | Prisma + libSQL（既存 DB にセッションテーブルを追加） |
| セッション有効期限 | 7日間 |
| トークンリフレッシュ | Discord のリフレッシュトークンを使用 |
| ログアウト | セッション破棄 + Cookie 削除 |

### 権限チェック

各 API リクエストで以下を検証する:

1. セッションが有効であること
2. ユーザーが対象ギルドで ManageGuild 権限またはオーナーであること
3. Bot が対象ギルドに参加していること

**権限不足時の挙動:**

| 状況 | HTTP ステータス | 画面表示 |
| --- | --- | --- |
| 未ログイン | 401 | ログイン画面にリダイレクト |
| 権限なし | 403 | 「権限がありません」メッセージ |
| Bot 未参加 | 403 | 「Bot をサーバーに追加してください」メッセージ + 招待リンク |

## 画面詳細

### 1. ランディングページ (`/`)

未ログイン時に表示。Bot の紹介と「Discord でログイン」ボタン。

| 要素 | 内容 |
| --- | --- |
| ロゴ | Ayasono ロゴ |
| キャッチコピー | 「サーバーに彩りを加え、豊かで快適なコミュニティを」 |
| ログインボタン | 「Discord でログイン」→ `/login` へ遷移 |

---

### 2. サーバー選択画面 (`/servers`)

ログイン後に表示。管理権限を持つサーバーをグリッド表示。

| 要素 | 内容 |
| --- | --- |
| サーバーカード | サーバーアイコン + サーバー名 |
| Bot 参加状態 | 参加済み → 「管理する」ボタン / 未参加 → 「Bot を追加」ボタン |
| フィルター | Bot 参加済みのみ表示する切り替え |

**表示条件:**
- ユーザーが ManageGuild 権限またはオーナーであるギルドのみ表示

---

### 3. ダッシュボード (`/servers/:guildId`)

サーバーの概要と各機能の有効/無効状態を一覧表示。

| セクション | 内容 |
| --- | --- |
| サーバー情報 | サーバー名・アイコン・メンバー数 |
| 機能ステータス一覧 | 各機能の有効/無効バッジ + 設定ページへのリンク |

**機能ステータスカード:**

| 表示項目 | 内容 |
| --- | --- |
| 機能名 | AFK、Bump リマインダー等 |
| 状態バッジ | 有効（緑）/ 無効（グレー）/ 未設定（黄） |
| 概要情報 | 機能ごとの要約（例: 「メンション: @Bumper」） |
| リンク | 「設定する →」で各設定ページへ |

---

### 4. サーバー設定 (`/servers/:guildId/general`)

| 設定項目 | UIコンポーネント | 値 |
| --- | --- | --- |
| 言語 | セレクトボックス | `ja` / `en` |
| エラー通知チャンネル | チャンネルセレクター | テキストチャンネル一覧 |

**操作:**
- 設定リセット（デフォルトに戻す）ボタン
- 全機能リセット（すべての機能設定を初期化）ボタン — 確認ダイアログ必須

**インポート/エクスポート:**

| 操作 | UIコンポーネント | 説明 |
| --- | --- | --- |
| エクスポート | ボタン | 現在の全設定を JSON ファイルとしてダウンロード |
| インポート | ファイルアップロード | JSON ファイルを選択してアップロード |

インポートフロー:
1. JSON ファイル選択
2. バリデーション結果の表示（エラーがあれば詳細表示）
3. 変更内容のプレビュー（差分表示）
4. 確認ダイアログ → 適用

---

### 7. VC 募集設定 (`/servers/:guildId/vc-recruit`)

| 設定項目 | UIコンポーネント | 値 |
| --- | --- | --- |
| 有効/無効 | トグルスイッチ | Boolean |
| メンションロール | ロールセレクター（複数） | サーバーロール一覧（最大25） |

**セットアップ一覧（テーブル表示）:**

| 列 | 内容 |
| --- | --- |
| カテゴリ | Discord カテゴリ名（null の場合「サーバー全体」） |
| パネルチャンネル | パネル設置先チャンネル名 |
| 投稿チャンネル | 募集投稿先チャンネル名 |
| アーカイブ期間 | スレッド自動アーカイブ時間 |
| 操作 | 編集・削除ボタン |

**セットアップ新規作成フロー:**

1. 「+ セットアップを追加」ボタンをクリック
2. 作成ダイアログが表示される:
   - カテゴリ: カテゴリセレクター（任意、未選択で「サーバー全体」）
   - パネルチャンネル: チャンネルセレクター（必須、テキストチャンネル）
   - 投稿チャンネル: チャンネルセレクター（必須、テキストチャンネル）
   - アーカイブ期間: セレクトボックス（60分 / 24時間 / 3日 / 7日、デフォルト: 24時間）
3. 「作成」ボタン → Discord にパネルメッセージを送信 + DB 保存

**制約:**
- メンションロール上限: 25（Discord SelectMenu の制限）

**操作:**
- 設定リセット（確認ダイアログ付き）

### 11. チケット設定 (`/servers/:guildId/tickets`)

#### カテゴリ一覧画面

| 要素 | 内容 |
| --- | --- |
| カテゴリカード | カテゴリ名・有効状態・チケット数 |
| 新規作成ボタン | 「+ パネルを作成」 |

#### パネル新規作成フロー

1. 「+ パネルを作成」ボタンをクリック
2. 作成ダイアログが表示される:
   - カテゴリ: カテゴリセレクター（必須）
   - パネル設置チャンネル: チャンネルセレクター（必須、テキストチャンネル）

   - パネルタイトル: テキスト入力（デフォルト: 「サポート」）
   - パネル説明文: テキストエリア（デフォルト値あり）
   - パネルカラー: カラーピッカー（デフォルト: `#00A8F3`）
3. Embed ライブプレビューで確認
4. 「作成」ボタン → 確認ダイアログ → Discord にパネルメッセージを送信 + DB 保存

#### カテゴリ詳細画面

**パネル設定:**

| 設定項目 | UIコンポーネント | 値 |
| --- | --- | --- |
| パネルタイトル | テキスト入力 | デフォルト: 「サポート」 |
| パネル説明文 | テキストエリア | デフォルト値あり |
| パネルカラー | カラーピッカー | HEX（デフォルト: `#00A8F3`） |

**Embed ライブプレビュー:**
- パネル設定の変更がリアルタイムで Discord 風プレビューに反映

**運用設定:**

| 設定項目 | UIコンポーネント | 値 |
| --- | --- | --- |
| スタッフロール | ロールセレクター（複数） | サーバーロール一覧 |
| 自動削除日数 | 数値入力 | 1〜90（デフォルト: 7） |
| 最大チケット数/ユーザー | 数値入力 | デフォルト: 1 |

**チケット一覧（読み取り専用）:**

| 列 | 内容 |
| --- | --- |
| チケット番号 | `#001` 等 |
| 件名 | ユーザーが入力した件名 |
| 作成者 | ユーザー名 |
| ステータス | `open` / `closed` |
| 作成日時 | タイムスタンプ |

**操作:**
- パネル作成（Discord にメッセージ送信）
- パネル編集（Embed 内容更新）
- パネル削除（確認ダイアログ付き）

## 共通 UI コンポーネント

### Discord リソースセレクター

Discord のチャンネル・ロール・ユーザー・カテゴリを選択するための共通コンポーネント。

**チャンネルセレクター:**

| 仕様 | 内容 |
| --- | --- |
| 表示形式 | 検索可能ドロップダウン |
| フィルター | チャンネル種別（テキスト/ボイス）で絞り込み |
| 表示情報 | `#チャンネル名`（テキスト）/ `🔊 チャンネル名`（ボイス） |
| データ取得 | Discord API 経由（Bot トークンで取得、API でキャッシュ） |

**カテゴリセレクター:**

| 仕様 | 内容 |
| --- | --- |
| 表示形式 | 検索可能ドロップダウン |
| 表示情報 | `📁 カテゴリ名` |
| 用途 | チケットのパネル作成、VC 募集のセットアップ |
| データ取得 | Discord API 経由（Bot トークンで取得、API でキャッシュ） |

**ロールセレクター:**

| 仕様 | 内容 |
| --- | --- |
| 表示形式 | 検索可能ドロップダウン（複数選択対応） |
| 表示情報 | ロール名 + ロールカラーのドット |
| 除外 | `@everyone` と Bot マネージドロールは非表示 |

**ユーザーセレクター:**

| 仕様 | 内容 |
| --- | --- |
| 表示形式 | 検索可能ドロップダウン（複数選択対応） |
| 表示情報 | アバター + ユーザー名 |
| データ取得 | サーバーメンバー一覧（ページネーション対応） |

### Embed エディタ（ライブプレビュー付き）

チケット・リアクションロール・メッセージ固定で使用する共通 Embed 編集コンポーネント。

**エディタ（左側）:**

| フィールド | UIコンポーネント | 必須 |
| --- | --- | --- |
| タイトル | テキスト入力 | ❌ |
| 説明文 | テキストエリア（Markdown 対応） | ❌ |
| カラー | カラーピッカー（HEX 入力対応） | ❌ |
| フィールド（配列） | 名前 + 値 + インライン切替 | ❌ |
| 著者名 | テキスト入力 | ❌ |
| 著者アイコン URL | テキスト入力 | ❌ |
| サムネイル URL | テキスト入力 | ❌ |
| 画像 URL | テキスト入力 | ❌ |
| フッターテキスト | テキスト入力 | ❌ |
| フッターアイコン URL | テキスト入力 | ❌ |

**プレビュー（右側）:**

- Discord メッセージ風の見た目でリアルタイム表示
- カラーは左端のアクセントバーに反映
- Markdown はレンダリングして表示
- フィールドのインライン配置も再現

**レイアウト:**
- PC: エディタとプレビューを左右に並べて表示
- モバイル: エディタの下にプレビューを配置（縦並び）

### カラーピッカー

| 仕様 | 内容 |
| --- | --- |
| 入力方式 | ビジュアルピッカー + HEX テキスト入力 |
| デフォルト | `#00A8F3` |
| プリセット | よく使う色をクイック選択 |

### 保存 UI

| 仕様 | 内容 |
| --- | --- |
| 保存方式 | 明示的な「保存」ボタン（自動保存しない） |
| 未保存検知 | 変更があれば画面下部にフローティングバーで「未保存の変更があります」+ 保存/破棄ボタン |
| 保存確認 | パネル作成・削除など破壊的操作は確認ダイアログ表示 |
| フィードバック | 保存成功 → トースト通知 / 失敗 → エラーメッセージ表示 |

**保存時バリデーション:**

設定の保存時に不整合な設定を検出し、エラーとして表示する。エラーがある場合は保存を実行しない。

| 機能 | 不整合条件 | エラーメッセージ |
| --- | --- | --- |
| AFK | 有効 + チャンネル未設定 | AFK チャンネルを設定してください |
| VAC | 有効 + トリガーチャンネル0件 | トリガーチャンネルを1つ以上追加してください |
| メンバーログ | 有効 + ログチャンネル未設定 | ログチャンネルを設定してください |
| VC 募集 | 有効 + セットアップ0件 | セットアップを1つ以上追加してください |
| チケット | スタッフロール未設定 | スタッフロールを1つ以上設定してください |

### トースト通知

| 種別 | 色 | 用途 |
| --- | --- | --- |
| 成功 | 緑 | 保存成功、操作完了 |
| エラー | 赤 | 保存失敗、バリデーションエラー |
| 警告 | 黄 | 設定不整合、注意喚起 |
| 情報 | 青 | 一般的な通知 |

### ローディング状態

| 状態 | 表示 |
| --- | --- |
| 初回データ取得中 | スケルトンスクリーン（コンテンツ領域にプレースホルダー表示） |
| 保存処理中 | 保存ボタンをローディング表示 + 操作無効化 |
| パネル作成/削除中 | モーダル内にスピナー表示 + 操作無効化 |

### エラー状態

| 状態 | 表示 |
| --- | --- |
| データ取得失敗 | エラーメッセージ +「再試行」ボタン |
| セッション切れ | 「セッションが切れました」トースト → ログイン画面にリダイレクト |
| Bot 権限不足 | 不足している権限名を表示 +「Bot の権限を確認してください」メッセージ |
| Discord API エラー | 「Discord との通信に失敗しました」+「再試行」ボタン |
| ネットワークエラー | 「ネットワーク接続を確認してください」メッセージ |

### レスポンス形式

**成功:**
```json
{
  "data": { ... }
}
```

**エラー:**
```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "説明テキスト",
    "details": { ... }
  }
}
```

**エラーコード一覧:**

| コード | HTTP | 説明 |
| --- | --- | --- |
| `UNAUTHORIZED` | 401 | 未認証 |
| `FORBIDDEN` | 403 | 権限不足 |
| `NOT_FOUND` | 404 | リソース未存在 |
| `VALIDATION_ERROR` | 400 | バリデーションエラー |
| `CONFLICT` | 409 | リソース競合（重複登録等） |
| `RATE_LIMITED` | 429 | レートリミット超過 |
| `DISCORD_API_ERROR` | 502 | Discord API エラー |
| `INTERNAL_ERROR` | 500 | 内部エラー |

## セキュリティ要件

| 項目 | 対策 |
| --- | --- |
| 認証 | Discord OAuth2 + HttpOnly Cookie セッション |
| CSRF | CSRF トークンまたは SameSite Cookie |
| XSS | React のデフォルトエスケープ + CSP ヘッダー |
| レートリミット | API エンドポイント種別ごとにレート制限（下記参照） |
| 入力バリデーション | Zod スキーマによるサーバーサイドバリデーション |
| CORS | 本番環境では許可オリジンを明示的に制限（既存実装済み） |
| セキュリティヘッダー | `X-Frame-Options`, `X-Content-Type-Options`, `Strict-Transport-Security` |
| 権限チェック | 全 API で ManageGuild 権限を検証（ミドルウェアレベル） |

### レートリミット

**エンドポイント種別ごと:**

| 種別 | 対象 | 制限値 | 理由 |
| --- | --- | --- | --- |
| 認証系 | `/api/auth/*` | 5回/分/IP | ブルートフォース防止 |
| 読み取り系 | GET `/api/guilds/*` | 30回/分/ユーザー | 画面遷移で複数リクエストが飛ぶため余裕を持たせる |
| 書き込み系 | PATCH, POST, DELETE | 10回/分/ユーザー | 設定保存の連打防止 |
| Discord 連携系 | パネル作成・削除 | 5回/分/ユーザー | Discord API 側のレート制限に合わせて控えめに |
| インポート | `/api/guilds/:id/config/import` | 3回/分/ユーザー | 重い処理のため |

**グローバルリミット:**

| 対象 | 制限値 | 理由 |
| --- | --- | --- |
| 全 API 合算/ユーザー | 60回/分 | 通常操作で超えない上限 |
| 全 API 合算/IP | 120回/分 | 同一 IP から複数ユーザーがアクセスする場合を考慮 |

**レスポンス:** 制限超過時は `429 Too Many Requests` + `Retry-After` ヘッダーを返却

## 制約・制限事項

| 制約 | 値 | 理由 |
| --- | --- | --- |
| メンションロール上限（VC 募集） | 25 | Discord SelectMenu の制限 |
| ボタン上限/行（リアクションロール） | 5 | Discord ActionRow の制限 |
| ボタン上限/メッセージ（リアクションロール） | 25 | Discord メッセージの制限 |
| チケット自動削除日数範囲 | 1〜90日 | 運用上の妥当な範囲 |
| スレッドアーカイブ期間 | 60 / 1440 / 4320 / 10080 分 | Discord の固定選択肢 |
| Embed 合計文字数 | 6000文字 | Discord の制限 |
| Embed タイトル | 最大256文字 | Discord の制限 |
| Embed 説明文 | 最大4096文字 | Discord の制限 |
| Embed フィールド数 | 最大25 | Discord の制限 |
| Embed フィールド名 | 最大256文字 | Discord の制限 |
| Embed フィールド値 | 最大1024文字 | Discord の制限 |
| Embed フッター | 最大2048文字 | Discord の制限 |
| Embed 著者名 | 最大256文字 | Discord の制限 |
| メンバーログ メッセージテンプレート | 最大500文字 | MEMBER_LOG_SPEC 準拠 |

## 依存関係

| 依存先 | 内容 |
| --- | --- |
| Discord OAuth2 API | ユーザー認証・ギルド情報取得 |
| Discord Bot API | チャンネル/ロール/メンバー情報取得、パネルメッセージの送信・編集・削除 |
| 既存 Prisma スキーマ | 全設定データの読み書き |
| 既存 ConfigService 群 | 設定のバリデーション・デフォルト値・ビジネスロジック |
| 既存 Fastify サーバー | HTTP サーバー基盤（ルーティング・ミドルウェア） |
