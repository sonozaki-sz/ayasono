# メッセージ削除機能 - 仕様書

> Message Delete Function - モデレーター向けメッセージ一括削除

最終更新: 2026年3月11日

## 📋 概要

### 目的

モデレーター権限を持つユーザーが、サーバー内の全チャンネルまたは特定チャンネルにわたって、特定ユーザーのメッセージ・キーワード一致メッセージ・指定件数のメッセージを一括削除できる機能を提供し、スパムや不適切なメッセージの迅速な対応を可能にします。

### 主な用途

1. **スパム対策**: サーバー全体に渡るスパムメッセージを横断的に一括削除
2. **不適切なコンテンツの削除**: 特定ユーザーの違反メッセージをサーバー全体から一括削除
3. **キーワード削除**: 特定の言葉・フレーズを含むメッセージをサーバー全体から削除
4. **チャンネルクリーンアップ**: テストメッセージや古いメッセージの整理
5. **モデレーション効率化**: 手動削除の手間を大幅に削減

### 特徴

- **サーバー全チャンネル対応**: チャンネルを指定しない場合、サーバー内の全テキストチャンネルを横断して削除
- **柔軟な削除条件**: 件数・ユーザー・Bot/Webhook・キーワード・期間（日数/日付範囲）・チャンネルを自由に組み合わせ
- **期間指定削除**: 過去N日以内（`days`）や特定の日付範囲（`after` / `before`）でメッセージを絞り込んで削除
- **14日以上前のメッセージにも対応**: 古いメッセージも個別削除で対応
- **安全性重視**: フィルタ条件（`count`・`user`・`keyword`・`days`・`after`・`before`）のいずれも未指定時は実行を拒否
- **スキャン進捗表示**: チャンネルスキャン中もリアルタイムで進捗を表示（スキャン済み総件数・一致件数/削除上限件数）
- **削除進捗表示**: 大量削除時にリアルタイムで合計進捗とチャンネルごとの削除件数を表示
- **処理中ロック**: コマンド実行中（Phase 1〜3）の重複実行をサーバー単位で防止
- **実行確認ダイアログ**: 削除実行前に確認ダイアログを表示。誤操作を防止（スキップ不可）

---

## 🎯 主要機能

### `/message-delete` コマンド

サーバー全体のメッセージを一括削除

---

## ⚙️ コマンド仕様

### オプション

**1. `count` (オプション)**

- 削除するメッセージ数（全チャンネル合計の上限件数）
- 範囲: 1〜1000
- 未指定の場合: 全チャンネル合計で最新1000件を上限として削除

**2. `user` (オプション)**

- 削除対象のユーザーをユーザーID またはメンション（`<@ID>`）で指定
- 指定した場合、そのユーザーのメッセージのみ削除
- **通常ユーザー・Bot**: `@` を入力するとDiscordの補完が効く（メンション形式で選択可能）
- **Webhook**: Discordの補完に表示されないため、Webhook IDを手入力する必要がある
- 受け付け形式: 生のID（17〜20桁の数字）または `<@ID>` 形式のメンション
- 不正な形式を入力した場合はエラー
- 未指定の場合、すべてのユーザーのメッセージが対象

**3. `keyword` (オプション)**

- メッセージの本文に対して部分一致検索を行い、マッチしたメッセージのみを削除対象とする
- 大文字/小文字を区別しない（case-insensitive）
- `user` オプションと併用可（条件の AND として機能）
- 未指定の場合、テキスト内容によるフィルタリングは行わない

**4. `days` (オプション)**

- 過去N日以内に投稿されたメッセージのみを削除対象とする
- 範囲: 1〜366（約1年間）
- 例: `days:7` → 過去7日以内のメッセージのみ削除
- `after` / `before` との同時指定不可（排他）
- `count` と組み合わせ可（期間内でさらに件数を絞る）

**5. `after` (オプション)**

- この日時以降に投稿されたメッセージのみを削除対象とする
- 受け付け形式:
  - `YYYY-MM-DD` → ロケール推定タイムゾーンの `00:00:00` として解釈
  - `YYYY-MM-DDTHH:MM:SS` → ロケール推定タイムゾーンとして解釈
  - `YYYY-MM-DDTHH:MM:SS±HH:MM` → 指定オフセットをそのまま適用
- タイムゾーン指定がない場合は `interaction.locale` から代表タイムゾーンを推定して解釈する（日本語 → JST、UTC 系言語 → UTC など）
- 例: `after:2026-01-01`（ロケール: 日本語）→ 2026年1月1日 00:00:00 JST 以降のメッセージのみ削除
- `days` との同時指定不可（排他）
- `before` と組み合わせることで日付範囲を指定可能
- 未来の日付は指定不可（`YYYY-MM-DD` 形式はその日の `00:00:00 ロケール時刻` が現在以前であれば有効。当日指定は許可）

**6. `before` (オプション)**

- この日時以前に投稿されたメッセージのみを削除対象とする
- 受け付け形式:
  - `YYYY-MM-DD` → ロケール推定タイムゾーンの `23:59:59` として解釈（当日末尾まで含める）
  - `YYYY-MM-DDTHH:MM:SS` → ロケール推定タイムゾーンとして解釈
  - `YYYY-MM-DDTHH:MM:SS±HH:MM` → 指定オフセットをそのまま適用
- タイムゾーン指定がない場合は `interaction.locale` から代表タイムゾーンを推定して解釈する（日本語 → JST、UTC 系言語 → UTC など）
- 例: `before:2026-02-01`（ロケール: 日本語）→ 2026年2月1日 23:59:59 JST 以前のメッセージのみ削除
- `days` との同時指定不可（排他）
- `after` と組み合わせることで日付範囲を指定可能
- 未来の日付は指定不可。ただし、`YYYY-MM-DD` 形式の当日指定は例外として許可する（解釈上の時刻 `23:59:59.999 UTC` が将来であっても有効）
  - `YYYY-MM-DD` 形式: その日の `00:00:00 UTC` が現在以前かどうかで判定（当日指定は有効）
  - `YYYY-MM-DDTHH:MM:SS[±HH:MM]` 形式: 指定時刻そのものが現在以前かどうかで判定

**7. `channel` (オプション)**

- 削除対象を絞り込むチャンネルを指定
- **未指定の場合、サーバー内の全テキストチャンネルを対象** とする
- DM 以外の全テキストベースチャンネル（TextChannel / NewsChannel / Thread / VoiceChannel ※VC はテキストチャット機能を持つため対象）が対象
- 指定した場合にBotがアクセス権を持たないチャンネルはエラーメッセージを返して終了する
- 未指定（全チャンネル走査）の場合にBotがアクセス権を持たないチャンネルはスキップする

### バリデーション

- 以下の「フィルタ条件オプション」のいずれも未指定の場合は実行を拒否する：
  - `count`・`user`・`keyword`・`days`・`after`・`before`
  - `channel` のみ指定でも拒否（フィルタ条件なしでチャンネル全削除になるため）
- 指定した `channel` にBotがアクセスできない場合はエラー（`channel` 未指定の全チャンネル走査時はスキップ）
- `days` と `after` / `before` を同時指定した場合はエラー（排他）
- `after` >= `before`（開始が終了と同じまたは後）の場合はエラー
- `after` / `before` に無効な日付形式を指定した場合はエラー
- `after` / `before` に未来の日付を指定した場合はエラー
  - `after` の `YYYY-MM-DD` 形式: その日の `00:00:00 ロケール時刻` が現在以前かどうかで判定（当日指定は有効）
  - `before` の `YYYY-MM-DD` 形式: その日の `00:00:00 ロケール時刻` が現在以前かどうかで判定（当日指定は例外として許可。解釈上の `23:59:59 ロケール時刻` が将来であっても有効）
  - `YYYY-MM-DDTHH:MM:SS[±HH:MM]` 形式（`after` / `before` 共通）: 指定時刻そのものが現在以前かどうかで判定

## ⚠️ 実行確認ダイアログ

### 概要

オプション検証・権限チェック・ロック取得・スキャンフェーズを経た後、スキャン結果を元に確認ダイアログを Ephemeral で表示する。スキップ設定はなく、常に表示される。ユーザーが承認するまで削除処理は開始しない。

### ダイアログ UI

確認ダイアログは**Embed 2枚**（コマンド条件 Embed + 削除対象メッセージ Embed）で構成する。フィルターや除外操作でプレビューを調整できる。

- **1ページあたり5件**（`MSG_DEL_PAGE_SIZE = 5` と共通）
- フィルター（投稿者・days/after/before・キーワード）でプレビューを絞り込み可能
- 現在のページに表示中のメッセージを選択して**削除対象から除外**できる
- 除外したメッセージは Embed 内で ~~取り消し線~~ 表示になり、件数から差し引かれる

> **実装メモ**: 確認ダイアログ表示前にメッセージを事前スキャンする必要がある。スキャン中は `スキャン中... N件 / 対象メッセージを検索中... #### Embed 構成

プレビューダイアログは **Embed 2 枚**で構成する。

**Embed 1: コマンド条件（静的）**

| 要素   | 内容                                                                                                                      |
| ------ | ------------------------------------------------------------------------------------------------------------------------- |
| title  | `📋 コマンド条件`                                                                                                         |
| color  | グレー（`#95a5a6`）                                                                                                       |
| fields | コマンドオプションと1:1。フィールド名はオプション名（ローカライズなし）。全オプションを常に表示し、未設定は括弧書きで明示 |

| フィールド名 | 設定ありの表示例  | 未設定の表示例            |
| ------------ | ----------------- | ------------------------- |
| `count`      | `500 件`          | `(上限なし: 1000 件)`     |
| `user`       | `<@ユーザーID>`   | `(全員対象)`              |
| `keyword`    | `"スパム"`        | `(なし)`                  |
| `days`       | `過去 7 日間`     | — （after/before と排他） |
| `after`      | `2026-01-01 以降` | `(なし)`                  |
| `before`     | `2026-02-01 以前` | `(なし)`                  |
| `channel`    | `<#チャンネルID>` | `(サーバー全体)`          |

- フィルター操作中も変化しない（静的）
- `days` または `after`/`before` の設定された方のフィールドのみ表示し、もう一方は省略する

**Embed 2: 削除対象メッセージ（動的）**

| 要素   | 内容                                                                                                                                                                                                                      |
| ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| title  | `📋 削除対象メッセージ（N / T ページ）`                                                                                                                                                                                   |
| fields | メッセージ1件ずつ（5件/ページ）。除外済みはフィールド名を `~~...~~` で表示。メッセージ本文はEmbedフィールドの文字数上限に収まるよう省略し、省略した場合は末尾に `…` を付加する（`MSG_DEL_CONTENT_MAX_LENGTH` 定数で制御） |

> **タイムスタンプ表示**: メッセージの投稿日時は Discord の `<t:unix秒:f>` 形式で表示する（例: `<t:1740912600:f>`）。Discord クライアントが閲覧ユーザーのローカル時刻に自動変換して表示するため、Bot のサーバータイムゾーンに依存しない。Embed のビジュアル例（`2026/02/25 10:30` など）はドキュメント上の可読性のための表記であり、実装上は `<t:...:f>` 形式を使用する。

> フィルター条件は各フィルターボタンのラベル・スタイル（色）で確認できるため、別途 Embed は設けない。フィルター未適用は `ButtonStyle.Secondary`（グレー）、フィルター適用中は `ButtonStyle.Primary`（青）で表示する。

#### 除外の仕様

- 除外セットはページをまたいで保持される
- 除外済みメッセージは Embed のフィールド名を ~~取り消し線~~ で表示
- 行4のセレクトメニューは現在ページを表示するたびに**除外済みの選択肢を選択済み状態**で表示
- 「削除する」ボタンのラベルは `🗑️ 削除する（N件）`（除外分を差し引いた件数）
- 全件除外した状態では「削除する」ボタンを無効化（disabled）する
- **フィルターは表示の絞り込みのみ**であり、削除対象件数には影響しない。「削除する（N件）」の N はフィルター状態によらず `スキャン結果全体 − 除外セット件数` で一定となる。フィルターをかけた状態で「削除する」を押しても、表示されていないメッセージも含めて全件削除される（フィルターは確認のための絞り込み表示であり、削除範囲を変更するものではない）

### 最終確認ダイアログ

| 行  | コンポーネント                           | customId                                                                       | 動作                                                                        |

| 2   | ボタン×3（削除・設定し直す・キャンセル） | `msgdel_final_yes` / `msgdel_final_back` / `msgdel_final_no`                   | 削除実行（**Danger スタイル**）/ プレビューへ戻る / 終了                    |

- フィルター・除外操作はできない（読み取り専用の確認画面）
- 「◀ 設定し直す」を押すとプレビューダイアログに戻る（除外セット・フィルター状態は保持）
- 「❌ キャンセル」を押すと操作を終了する
- タイムアウトはプレビューダイアログと共通の **14分**タイムアウト

### タイムアウト設計

コマンド全体の処理は、Discord Interaction token の有効期限（15分）を考慮して **3フェーズ**に分割し、それぞれ独立したタイムアウトを持つ。

```
Phase 1: コマンド実行 ─────────────── スキャン完了
Phase 2: スキャン完了 ─────────────── 削除実行ボタン押下
Phase 3: 削除実行ボタン押下 ────────── 削除完了
```

| フェーズ | Interaction token                           | タイムアウト                             | タイムアウト時の動作                                                                |
| -------- | ------------------------------------------- | ---------------------------------------- | ----------------------------------------------------------------------------------- |
| Phase 1  | `deferReply` 後の元コマンド token           | **14 分**                                | スキャンを中断し、収集済みがあればタイムアウト通知後にプレビューへ遷移。0件なら終了 |
| Phase 2  | Phase 1 完了時点で新たに 14 分計測開始      | **14 分**（プレビュー + 最終確認で共有） | コンポーネントをすべて削除し、タイムアウトメッセージを表示して終了                  |
| Phase 3  | 削除実行ボタンの `deferUpdate()` 済み token | **14 分**                                | タイムアウトメッセージを表示して終了（削除済み件数を通知）                          |

**設計上の注意点:**

- Phase 2 のプレビューと最終確認は共通の 14 分タイムアウトを持つ。プレビューで消費した時間は最終確認の残り時間から差し引かれる（合計で 14 分以内）。
- 各タイムアウト値は Discord API の 15 分制限に対して **1 分のバッファ**を設けている（タイムアウト通知メッセージ self を確実に送信するため）。
- Phase 2 のモーダル操作（ページジャンプ・フィルター入力）は各ボタン interaction の fresh な token を使用するため、Phase 1 の token 失効の影響を受けない。

## 💡 使用例

### ⚠️ 注意: フィルタ条件オプションの全未指定は禁止

```
/message-delete
```

- **エラー**: フィルタ条件（`count`・`user`・`keyword`・`days`・`after`・`before`）がすべて未指定の場合は警告を表示して中止

```
/message-delete channel:#general
```

- **エラー**: `channel` のみ指定も禁止（フィルタ条件なしでチャンネル全削除になるため）

```
/message-delete days:7 after:2026-01-01
```

- **エラー**: `days` と `after` / `before` の同時指定は不可

```
/message-delete after:2026-02-28 before:2026-01-01
```

- **エラー**: `after` が `before` と同じまたは後の日時は不可

### 例1: 特定ユーザーのメッセージをサーバー全体から削除

```
/message-delete user:@BadUser
```

- サーバー内の**全チャンネル**から `@BadUser` のメッセージをすべて検索して削除
- チャンネルごとに順次処理し、進捗を表示

### 例2: キーワードを含むメッセージをサーバー全体から削除

```
/message-delete keyword:スパム
```

- サーバー内の**全チャンネル**から「スパム」を含むメッセージをすべて削除

### 例3: ユーザー + キーワードの組み合わせで削除

```
/message-delete user:@BadUser keyword:宣伝
```

- サーバー内の全チャンネルから `@BadUser` が投稿した「宣伝」を含むメッセージを削除

### 例4: 件数を指定してサーバー全体から削除

```
/message-delete count:100 user:@Spammer
```

- サーバー全体から `@Spammer` のメッセージを最大100件削除
- 件数は全チャンネルの合計（例: チャンネルAで60件、チャンネルBで40件）

### 例5: 特定チャンネルに絞って削除

```
/message-delete keyword:広告 channel:#general
```

- `#general` チャンネルのみで「広告」を含むメッセージを削除

### 例6: 古いプロフィールメッセージを削除

```
/message-delete count:5 user:@User channel:#profile
```

- `#profile` チャンネルの `@User` のメッセージを5件削除
- 3ヶ月前のメッセージでも削除可能（個別削除で対応）

### 例7: 大量削除（サーバー全体 + 件数上限あり）

```
/message-delete count:500 keyword:NGワード
```

- サーバー全体から「NGワード」を含むメッセージを最大500件削除
- 100件ずつ複数回に分けて削除処理を実行

### 例8: 過去7日以内のメッセージを削除（日数指定）

```
/message-delete days:7 user:@Spammer
```

- サーバー全体から `@Spammer` の過去7日以内のメッセージをすべて削除

### 例9: 特定日以降のメッセージを削除（after 指定）

```
/message-delete after:2026-02-01 keyword:宣伝
```

- 2026年2月1日以降に投稿された「宣伝」を含むメッセージをサーバー全体から削除

### 例10: 日付範囲を指定して削除（after + before）

```
/message-delete after:2026-01-01 before:2026-01-31 user:@BadUser
```

- 2026年1月1日〜1月31日の間に `@BadUser` が投稿したメッセージをサーバー全体から削除

### 例11: 期間 + チャンネル絞り込み

```
/message-delete days:3 channel:#general
```

- `#general` チャンネルの過去3日以内のメッセージをすべて削除

### 例12: 件数上限 + 日付範囲

```
/message-delete count:200 after:2026-02-20T00:00:00 before:2026-02-27T23:59:59
```

- 指定期間内のメッセージを最大200件削除（時刻まで指定する例）

## 🔒 権限チェック

### 実行者の必要権限

- `MANAGE_MESSAGES` (メッセージ管理)

### Botの必要権限

- `MANAGE_MESSAGES` (メッセージ管理)
- `READ_MESSAGE_HISTORY` (メッセージ履歴の閲覧)
- `VIEW_CHANNEL` (チャンネルの閲覧)

**権限不足の場合:**

- エラーメッセージを表示
- ログに記録

### 処理中ロック

- **ロック対象**: コマンド実行全期間（Phase 1〜3）
- **ロックスコープ**: サーバー単位（同一サーバー内のどのユーザーがロック中でも新たなコマンド実行を拒否する）
- ロック中に実行しようとした場合はコマンド実行時点で即座に拒否する（スキャンや削除実行後に弾かれることはない）
- **ロック解放タイミング**: 正常完了・キャンセル・タイムアウト・エラー・例外のすべての終了パスで `finally` ブロックにより確実に解放する
- **Bot 再起動時**: ロックはメモリ上で管理されるため、Bot が落ちて再起動した場合はすべてのロックが自動的にクリアされる

### 同時実行時の挙動

**同一サーバー内でのコマンド実行:**

- いずれかのユーザーがコマンド実行中（Phase 1〜3）の場合、他のユーザーを含む新たなコマンド実行はコマンド実行時点で拒否される

## 🔄 処理フロー

````
1. コマンド実行
   ↓
2. オプション検証
   - count・user・keyword・days・after・before がすべて未指定の場合は警告を出して中止
   - channel のみ指定でも中止
   - days と after/before の同時指定はエラー
   - after >= before の場合はエラー
   - after・before の日付形式が不正な場合はエラー
   - after・before に未来の日時を指定した場合はエラー
   ↓
3. 権限チェック
   - 実行者の権限確認
   - Botの権限確認
   ↓
4. 処理中ロック取得
   - 同一サーバー内でPhase 1〜3を実行中のユーザーがいる場合はエラーを返して終了
   ↓
5. 期間フィルターの計算
   - days 指定あり   → afterTs = Date.now() - days * 86400000, beforeTs = Infinity（未来メッセージは存在しないため制限なし）
   - after/before 指定 → 各日付文字列を Unix ミリ秒に変換
   - いずれも未指定   → afterTs = 0, beforeTs = Infinity（期間制限なし）
   ↓
6. 対象チャンネルリスト取得
   - channel 指定あり → そのチャンネルのみ（Botがアクセス権を持たない場合はエラーを返して終了）
   - channel 未指定   → サーバー内の全テキストチャンネル
                        （Botがアクセス不可のチャンネルはスキップ）
   ↓
7. 対象メッセージのプレスキャン（Phase 1: 14分タイムアウト）
   - 以下の進捗テキストをEphemeralで表示（「収集分を確認」ボタン付き）
     ```
     スキャン中... N件
     対象メッセージを検索中... M / L件
     ```
     （N=スキャン済み総件数、M=条件一致件数、L=削除上限件数 ＝ `count` 指定値、未指定時は 1000）
   - k-wayマージアルゴリズムで全チャンネルを横断し、最新順に条件一致メッセージを収集:
     1. `beforeTs` を Discord Snowflake ID に変換（初回フェッチ位置の最適化）
     2. 全チャンネルを**並列**で初期フェッチ（`limit: 100, before: beforeSnowflake`）し、
        チャンネルごとのカーソル（バッファ・最終メッセージID・消耗フラグ）を初期化
     3. k-wayマージループ（`収集件数 < count` の間）:
        a. キャンセルシグナル確認（「収集分を確認」ボタン押下時に中断）
        b. バッファが空かつ未消耗のチャンネルを追加フェッチ（直列・レートリミット考慮で 200ms 待機）
        c. 全チャンネルのバッファ先頭のうち最新タイムスタンプを持つメッセージを選択
        d. 選択できるメッセージがなければループ終了（全チャンネル消耗）
        e. user・keyword フィルタを適用し、条件不一致はスキップ（スキャン総件数 N にはカウント）
        f. 収集リストに追加
     4. チャンネル消耗判定: フェッチ結果が0件または100件未満、もしくは最古メッセージが
        `afterTs` より古い場合はそのチャンネルを消耗済みにマーク（以降フェッチしない）。
        `afterTs` より古いメッセージはバッファから除外する
     5. `count` 件収集した時点でループ早期終了（全チャンネルスキャン完了前でも停止）
   - 「収集分を確認」ボタンを押した場合：
     - 収集済メッセージが1件以上 → その結果で直接プレビューダイアログへ遷移（中間画面なし）
     - 収集済メッセージが0件 → 「❌ 削除をキャンセルしました。」を表示して終了
   - 14分タイムアウト時：
     - 収集済メッセージが1件以上 → `⌛ スキャンがタイムアウトしました。収集済みのメッセージでプレビューを表示します。` を表示してからプレビューダイアログへ遷移
     - 収集済メッセージが0件 → タイムアウトメッセージを表示して終了
   - 全チャンネルのスキャン完了1件以上 → 直接プレビューダイアログへ遷移（中間画面なし）
   - 全チャンネルのスキャン完了0件の場合 → 「ℹ️ 削除可能なメッセージが見つかりませんでした。」を表示して終了
   ↓
8. 実行確認ダイアログ（プレビュー + 最終確認：Phase 2: 14分タイムアウト・共有）
   - スキャン完了時点から 14 分のタイムアウトをセッション単位で計測開始
   - コマンド条件 Embed + 削除対象メッセージ Embed でプレビューを表示（ページネイション）
   - 「🗑️ 削除する」→ 最終確認ダイアログへ遷移（残り時間を引き継ぐ）
   - 「❌ キャンセル」→ キャンセルメッセージを表示して終了
   - タイムアウト → タイムアウトメッセージを表示して終了
   ↓
9. チャンネルごとにループ処理（プレスキャン結果を使用）
   ├── 9a. メッセージを14日以内/14日以上前に分類
   │    - 14日以内（新しい）: bulkDelete 対象
   │    - 14日以上前（古い）: 個別削除対象
   └── 9b. メッセージ削除（ループ）
        - 14日以内: 100件ずつ bulkDelete で高速削除
        - 14日以上前（古い）: 1件ずつ個別削除（レート制限考慮）
        - 以下の進捗テキストを更新しながら表示:
          ```
          削除中... N / T件
          #チャンネル名1: deleted / total件
          #チャンネル名2: deleted / total件
          ```
          （N=合計削除済み件数、T=合計対象件数）
   ↓
10. 全チャンネル処理完了
   ↓
11. 結果通知（Ephemeral）
   - 合計削除件数（チャンネル別内訳も表示）
   ↓
12. ログ記録
   - 実行者ID
   - 合計削除件数・削除したチャンネルID一覧
   - count（指定されている場合）
   - 対象ユーザーID（指定されている場合）
   - キーワード（指定されている場合）
   - 期間条件（days / after / before が指定されている場合）

※ ステップ 4〜12 のすべての終了パス（正常完了・キャンセル・タイムアウト・エラー・例外）で
  finally ブロックによりロックを解放する
````

## ⚠️ Discord API制限

### 14日制限（bulkDelete）

Discordの`bulkDelete`メソッドは、**14日以上前のメッセージを削除できません**。

**対応:**

- **14日以内**: `bulkDelete()`で高速一括削除（100件ずつ）
- **14日以上前（古い）**: `message.delete()`で個別削除（1件ずつ、時間がかかる）
- 削除対象のメッセージを自動的に分類して適切な方法で削除

**パフォーマンス:**

- 一括削除: 100件を数秒で削除
- 個別削除: 1件あたり約0.5秒（レート制限考慮）
  - 例: 100件の個別削除 → 約50秒
  - 例: 500件の個別削除 → 約250秒（約4分）

### レート制限

- 一度に削除できるメッセージ数: 最大100件（bulkDelete制限）
- メッセージフェッチ: 100件ずつ
- レート制限: 適切な間隔を空ける必要あり

**対応:**

- 100件ずつに分割してフェッチ・削除
- 大量削除時は進捗を表示

## 🛡️ エラーハンドリング

### 想定されるエラー

1. **フィルタ条件オプション（`count`・`user`・`keyword`・`days`・`after`・`before`）の全未指定**

   ```
   ⚠️ 警告: フィルタ条件（`count`・`user`・`keyword`・`days`・`after`・`before`）のいずれも未指定のため実行できません。
   少なくとも1つのフィルタ条件を指定してください。
   ```

2. **権限不足**

   ```
   ❌ この操作を実行する権限がありません。
   必要な権限: メッセージ管理
   ```

3. **Bot権限不足**

   ```
   ❌ Botにメッセージ削除権限がありません。
   ```

4. **メッセージが見つからない**

   ```
   ℹ️ 削除可能なメッセージが見つかりませんでした。
   ```

5. **メッセージが既に削除されている**
   - スキャンフェーズと削除フェーズの間に、対象メッセージがユーザー本人・別のBot・Discord側などによって削除された場合
   - メッセージが既に削除されているエラーは無視して処理を継続
   - ログに警告を記録（削除失敗件数は含めない）

6. **`days` と `after`/`before` の同時指定**

   ```
   ⚠️ `days` と `after`/`before` は同時に指定できません。どちらか一方を使用してください。
   ```

7. **`after` / `before` の日付形式が不正**

   ```
   ⚠️ `after` の日付形式が不正です。`YYYY-MM-DD` または `YYYY-MM-DDTHH:MM:SS` 形式で指定してください。
   ```

8. **`after` が `before` と同じまたは後**

   ```
   ⚠️ `after` は `before` より前の日時になるよう指定してください。
   ```

9. **`after` に未来の日付を指定した場合**

   ```
   ⚠️ `after` には現在以前の日付を指定してください。（当日の指定は有効です）
   ```

10. **`before` に未来の日付を指定した場合**

    ```
    ⚠️ `before` には現在以前の日付を指定してください。（当日の指定は有効です）
    ```

11. **実行確認のキャンセル**

    ```
    ❌ 削除をキャンセルしました。
    ```

12. **実行確認ダイアログのタイムアウト（14分）**

    ```
    ⌛ タイムアウトしました。再度コマンドを実行してください。
    ```

13. **処理中ロック（同一サーバーで別コマンドが実行中）**

    ```
    ⚠️ 現在このサーバーでメッセージ削除コマンドを実行中です。完了後に再度お試しください。
    ```

14. **指定した `channel` にBotがアクセスできない場合**

    ```
    ❌ 指定したチャンネルにアクセスできません。BotにReadMessageHistoryおよびManageMessages権限が必要です。
    ```

15. **削除処理のタイムアウト（Phase 3: 14分）**
    - 大量メッセージの削除中にDiscord Interaction tokenが失効した場合
    - N件はタイムアウト時点での削除済み件数

    ```
    ⌛ 削除処理がタイムアウトしました。削除済み: N件
    ```
