## Bumpリマインダー機能

### この機能でできること

[Disboard](https://disboard.org/) や [ディス速](https://dissoku.net/) の Bump コマンドを自動検知し、次に Bump できるようになる **2時間後**に自動でリマインダーを送信します。
「Bump の時間を忘れてしまう」「誰かが Bump してくれるのを待っている」という状況を解消します。

### 👤 メンバー向け — リマインダーを受け取る

Bump が実行されると、Bot からボタン付きのメッセージが届きます。

<table border="1" cellpadding="8" width="420">
<tr><th align="left">ℹ️ Bumpリマインダー機能</th></tr>
<tr><td>
54分後にリマインドが通知されます。
</td></tr>
<tr><td><kbd>🔔 ユーザー通知をONにする</kbd> &nbsp; <kbd>🔕 ユーザー通知をOFFにする</kbd></td></tr>
</table>

- **🔔 ユーザー通知をONにする** — 2時間後のリマインダーで自分に通知が届きます。既にONの場合でもそのまま成功します。
- **🔕 ユーザー通知をOFFにする** — 通知設定を解除します。既にOFFの場合でもそのまま成功します。

**💡 ヒント:** Bumpに協力したいときは「🔔 ユーザー通知をONにする」を押して通知をオンにしておきましょう。

---

### 🛡️ 管理者向け — 設定

#### リマインダーを有効化する

**デフォルトで有効**なので、通常は操作不要です。

```
/bump-reminder-config enable
```

#### リマインダーを無効化する

```
/bump-reminder-config disable
```

#### リマインダー時にメンションするロールを設定する

```
/bump-reminder-config set-mention role:@Bumpロール
```

> **📝 注:** ユーザー個別の通知設定はパネルのボタンから各自が行います。管理コマンドではロールのみ設定できます。

#### メンション設定を削除する

```
/bump-reminder-config remove-mention
```

#### 現在の設定を確認する

```
/bump-reminder-config view
```

**💡 ヒント:** `@Bump担当` のようなロールを作って設定しておくと、有志のメンバーだけにリマインダーが届くようにできます。

## メッセージ削除機能

### この機能でできること

**モデレーター向け**の機能です。条件を指定してサーバー内のメッセージを一括削除できます。

- 特定ユーザーのメッセージをサーバー全体から一掃（複数ユーザー指定可）
- 特定チャンネルに絞って削除（複数チャンネル指定可）
- キーワードを含むメッセージを絞り込んで削除
- 特定期間（日数・日付範囲）のメッセージを削除

> **⚠️ 必要な権限**: メッセージ管理（Manage Messages）またはサーバー管理者

---

### 🛡️ 基本的な使い方

#### コマンドオプション一覧

| オプション | 型     | 説明                                                                           |
| ---------- | ------ | ------------------------------------------------------------------------------ |
| `count`    | 数値   | 収集する件数の上限（省略時は最大1000件）                                       |
| `keyword`  | 文字列 | 指定キーワードを含むメッセージのみを対象（大文字小文字を区別しない）           |
| `days`     | 数値   | 過去 N 日以内のメッセージのみ対象（`after`/`before` とは排他）                 |
| `after`    | 日付   | この日時以降のメッセージのみ対象。 形式: `YYYY-MM-DD` or `YYYY-MM-DDTHH:MM:SS` |
| `before`   | 日付   | この日時以前のメッセージのみ対象。 形式: `YYYY-MM-DD` or `YYYY-MM-DDTHH:MM:SS` |

#### 条件設定ステップ

コマンド実行後、**条件設定画面**が表示されます。ここで対象ユーザーや対象チャンネルを選択できます。

| UI 要素                | 説明                                                                 |
| ---------------------- | -------------------------------------------------------------------- |
| ユーザー選択メニュー   | 削除対象のユーザーを選択（最大25人、複数選択可）。未選択で全ユーザー |
| チャンネル選択メニュー | 対象チャンネルを選択（最大25個、複数選択可）。未選択でサーバー全体   |
| Webhook ID 入力        | Webhook のメッセージを削除したい場合にモーダルから ID を入力         |
| スキャン開始           | 条件を確定してスキャンを開始                                         |
| キャンセル             | 操作を中止                                                           |

> **💡 ヒント**: コマンドオプション（`count`/`keyword`/`days`/`after`/`before`）と条件設定ステップ（ユーザー/チャンネル）を合わせて、最低1つのフィルター条件が必要です。

#### 使用例

```
# キーワードで絞り込み → 条件設定でチャンネルを選択
/message-delete keyword:スパム

# 最新30件を収集 → 条件設定でチャンネル・ユーザーを選択
/message-delete count:30

# 過去7日間のメッセージ → 条件設定でチャンネルを選択
/message-delete days:7

# 日付範囲を指定 → 条件設定でユーザーを選択
/message-delete after:2026-01-01 before:2026-01-31
```

#### 注意事項

- コマンドオプションと条件設定ステップを合わせて、最低1つのフィルター条件を指定してください
- `days` と `after`/`before` は同時に指定できません
- 条件設定ステップは3分以内に操作してください（タイムアウトで自動キャンセル）
- 14日以上前のメッセージは個別削除になるため、件数が多いと時間がかかります
- **この操作は取り消せません**

#### スキャン

コマンドを実行するとスキャンが開始され、進捗がリアルタイムで表示されます。

```
スキャン中... 1200件
対象メッセージを検索中... 12 / 1000件
```

スキャン完了を待たずに収集済みのメッセージでプレビューへ進むには「🔎 収集分を確認」ボタンを押してください。

#### プレビューダイアログ（Stage 1）

スキャン完了後、削除候補メッセージのプレビューが表示されます。

- **フィルター機能** — 投稿者・キーワード・日付範囲でプレビュー表示を絞り込めます（表示の切り替えのみで、削除対象は変わりません）
- **除外機能** — 削除したくない特定のメッセージを個別に除外できます
- **「削除する」** — 最終確認ダイアログ（Stage 2）へ進みます
- **「キャンセル」** — 操作を中止します

#### 最終確認ダイアログ（Stage 2）

削除対象のメッセージ一覧が再度表示されます。内容を確認してから削除を実行してください。

| ボタン        | 動作                       |
| ------------- | -------------------------- |
| 🗑 削除する   | 削除を実行して完了         |
| ◀ 戻る        | プレビューダイアログに戻る |
| ❌ キャンセル | 操作を中止する             |

#### 削除完了後

削除完了後に削除件数とチャンネル別の内訳が表示されます。

## AFK機能

### この機能でできること

VCに参加中のメンバーを、指定した「AFKチャンネル」へ素早く移動させる機能です。
「音声に反応がないメンバーを整理したい」「自分が少し離席したい」といった場面で活躍します。

### 👤 メンバー向け — 使い方

#### 自分をAFKチャンネルへ移動する

まず自分がボイスチャンネルに入った状態で、任意のテキストチャンネルに入力します。

```
/afk
```

これだけです。Bot がすぐに AFK チャンネルへ移動させてくれます。

#### 他のメンバーをAFKチャンネルへ移動する

```
/afk user:@ユーザー名
```

**実行例:**

```
/afk user:@田中さん
```

**💡 ヒント:** 少し席を外すときや、長時間応答がないメンバーを整理したいときに便利です。

**⚠️ 注意事項:**

- 対象のメンバーが VCに参加していないと移動できません。
- AFK機能が管理者によって有効化されている必要があります。
- AFKチャンネルが設定されていない場合は動作しません。

---

### 🛡️ 管理者向け — 初期設定

AFK 機能を使うには、まず管理者が AFK チャンネルを設定する必要があります。

#### AFKチャンネルを設定する

```
/afk-config set-channel channel:#AFKチャンネル名
```

これを実行すると AFK 機能が自動で有効になります。

#### 現在の設定を確認する

```
/afk-config view
```

設定中のAFKチャンネルと機能の有効/無効状態が表示されます。

## VC自動作成機能

### この機能でできること

「CreateVC」などの専用チャンネルに入ると、**自分だけの専用VC** が自動で作られます。
全員が退出すると、作られた VC は自動で削除されます。VC を手動で作る手間が省けます。

### 👤 メンバー向け — 使い方

#### VCを自動作成する

管理者が設定したトリガーチャンネル（例: `➕ CreateVC`）に入ります。
すると、あなたの専用VC（例: `そのざき's Room`）が自動で作成され、そこに移動されます。

#### 操作パネルでVCの設定を変更する

作成された VC のチャット欄に、**操作パネル**が自動で表示されます。
パネルのボタンを押すと、VC の設定を変更できます。

<table border="1" cellpadding="8" width="360">
<tr><th align="left">🎤 ボイスチャンネル操作パネル</th></tr>
<tr><td>
<kbd>✏️ VC名を変更</kbd><br>
<kbd>👥 人数制限を変更</kbd><br>
<kbd>🔇 メンバーをAFKに移動</kbd><br>
<kbd>🔄 パネルを最下部に移動</kbd>
</td></tr>
</table>

| ボタン                  | できること                              |
| ----------------------- | --------------------------------------- |
| ✏️ VC名を変更           | VCの名前を好きな名前に変更              |
| 👥 人数制限を変更       | 参加できる人数の上限を設定（0で無制限） |
| 🔇 メンバーをAFKに移動  | このVCのメンバーをAFKチャンネルへ送る   |
| 🔄 パネルを最下部に移動 | チャットが流れたときパネルを再表示      |

**⚠️ 注意:** パネルの操作はそのVCに参加中のメンバーのみ実行できます。

#### コマンドでVCの設定を変更する

パネルの代わりに、スラッシュコマンドでも設定変更できます。

```
# VC名を変更する
/vac vc-rename name:みんなのたまり場

# 人数制限を5人にする
/vac vc-limit limit:5

# 人数制限を無制限にする
/vac vc-limit limit:0
```

**⚠️ 注意:** これらのコマンドも、そのVCに参加中にのみ実行できます。

**💡 ヒント:**

- 名前を変えておくと、どんな集まりか一目でわかります。
- 少人数で話したいときは人数制限を設定しましょう。
- 全員が退出すると VC は自動で削除されます。

---

### 🛡️ 管理者向け — 設定

#### トリガーチャンネルを作成する

```
# カテゴリを指定して作成する
/vac-config create-trigger-vc category:カテゴリ名

# カテゴリなし（トップレベル）で作成する
/vac-config create-trigger-vc
```

実行すると「➕ CreateVC」という名前のVCが指定のカテゴリに作成されます。
このチャンネルにメンバーが入ると、自動でその人専用のVCが作られます。

#### トリガーチャンネルを削除する

```
/vac-config remove-trigger-vc category:カテゴリ名
```

#### 現在の設定を確認する

```
/vac-config view
```

## メッセージ固定機能

### この機能でできること

チャンネルのルールや重要なお知らせを、**常にチャンネルの一番下**に表示し続ける機能です。
Discord 標準の「ピン留め」は上に固定されますが、スティッキーメッセージは常に最新メッセージの直下に表示されるため見逃されにくくなります。

**主な用途の例:**

- チャンネルのルールやマナーを常に表示
- イベント告知や締め切りを見落とさないように
- 質問チャンネルのテンプレートを常時表示
- 注意事項をいつでも確認できるように

### 🛡️ 管理者向け — 使い方

#### スティッキーメッセージを設定する

**プレーンテキストで設定する:**

```
/sticky-message set channel:#チャンネル名
```

コマンドを実行すると、メッセージ内容を入力するポップアップが表示されます。

**Embed（カード形式）で設定する:**

```
/sticky-message set channel:#チャンネル名 style:embed
```

Embed で設定するとタイトル・説明文・カラーを設定でき、より見やすいメッセージになります。

**使用例:**

```
# ルールチャンネルにプレーンテキストでスティッキーメッセージを設定
/sticky-message set channel:#rules

# お知らせチャンネルにEmbed（緑色）で設定
/sticky-message set channel:#announcements style:embed
```

> コマンド実行後にポップアップが開きます。内容を入力して送信してください。

#### スティッキーメッセージを更新する

```
/sticky-message update channel:#チャンネル名
```

更新内容のポップアップが表示されます。変更したい項目だけ入力してください。

#### スティッキーメッセージを削除する

```
/sticky-message remove
```

コマンドを実行すると、設定済みのチャンネル一覧がセレクトメニューで表示されます。
削除したいチャンネルを選択し（複数選択可）、「🗑️ 削除する」ボタンを押して確定します。
Discord 上のメッセージも同時に削除されます。

#### 設定中のスティッキーメッセージを確認する

```
/sticky-message view
```

このサーバーで設定中のチャンネル一覧が表示されます。チャンネルを選ぶと詳細が確認できます。

**💡 ヒント:** ルールや告知を Embed 形式で作ると、色やタイトルで目を引きやすくなります。

## メンバーログ機能

### この機能でできること

サーバーへのメンバー参加・脱退を指定したチャンネルに自動で記録します。
誰がいつ参加・脱退したかを詳細な情報つきで通知します。

### 👤 メンバー向け — 通知の見方

参加・脱退があると、設定されたチャンネルに自動で通知が投稿されます。

**参加通知（緑色）に表示される情報:**

| フィールド         | 内容                                                       |
| ------------------ | ---------------------------------------------------------- |
| ユーザー           | 参加したメンバーのメンション                               |
| アカウント作成日時 | Discordアカウントの作成日時と経過年齢                      |
| サーバー参加日時   | このサーバーへの参加日時                                   |
| 招待元             | 使用された招待リンクと招待者（特定できない場合は「不明」） |
| メンバー数         | 参加後の現在メンバー数                                     |

**退出通知（赤色）に表示される情報:**

| フィールド         | 内容                                  |
| ------------------ | ------------------------------------- |
| ユーザー           | 退出したメンバーのメンション          |
| アカウント作成日時 | Discordアカウントの作成日時と経過年齢 |
| サーバー参加日時   | このサーバーへの参加日時              |
| サーバー退出日時   | 退出日時                              |
| 滞在期間           | サーバーへの滞在日数                  |
| メンバー数         | 退出後の現在メンバー数                |

---

### 🛡️ 管理者向け — 設定

#### ログチャンネルを設定する

```
/member-log-config set-channel channel:#メンバーログ
```

#### 機能を有効化する

```
/member-log-config enable
```

#### 機能を無効化する

```
/member-log-config disable
```

#### カスタム参加メッセージを設定する

```
/member-log-config set-join-message
```

コマンドを実行するとポップアップ（モーダル）が開き、**複数行のテキスト**を入力できます（最大500文字）。
以下のプレースホルダーが使用できます：

| プレースホルダー | 内容               |
| ---------------- | ------------------ |
| `{userMention}`  | ユーザーメンション |
| `{userName}`     | ユーザー名         |
| `{memberCount}`  | 現在のメンバー数   |
| `{serverName}`   | サーバー名         |

**入力例:**

```
ようこそ {userMention}さん！{serverName}へようこそ！
サーバーメンバーが {memberCount}人になりました 🎉
```

#### カスタム退出メッセージを設定する

```
/member-log-config set-leave-message
```

コマンドを実行するとポップアップ（モーダル）が開きます（最大500文字）。
プレースホルダーは参加メッセージと同じものが使用できます。

#### カスタム参加メッセージを削除する

```
/member-log-config clear-join-message
```

設定済みのカスタム参加メッセージを削除します。

#### カスタム退出メッセージを削除する

```
/member-log-config clear-leave-message
```

設定済みのカスタム退出メッセージを削除します。

#### 現在の設定を確認する

```
/member-log-config view
```

## VC募集機能

### この機能でできること

コマンドで専用チャンネルを自動作成し、VC参加者を募るメッセージをボタン＆モーダルで投稿する機能です。
「ゲームや雑談のVC、一緒に入る人いませんか？」という呼びかけを手間なく、フォーマット統一した形で行えます。

**主な特徴:**

- **チャンネル自動作成** — コマンド1つで募集作成・募集一覧チャンネルをセットで作成
- **ロール制限** — 不要な全体メンションを防ぐため、メンションに使えるロールは管理者が事前設定
- **スレッド自動作成** — 募集メッセージごとにスレッドが自動で立ち、参加者がそこ で返信できる
- **新規VC作成** — 既存VCへの参加のほか「新規VC作成」も選択可能
- **VCに参加ボタン** — 募集メッセージの「VCに参加」リンクボタンで対象VCへ直接ジャンプ
- **募集終了ボタン** — 募集メッセージの「🔇 募集終了」ボタンで新規作成VCを削除し、投稿パネルを「終了済み」状態に更新（投稿・スレッドは残る）
- **投稿削除ボタン** — 募集メッセージの「🗑️ 募集を削除」ボタンで投稿・スレッド・新規作成VCをまとめて削除

---

### 👤 メンバー向け — VC参加者を募集する

#### 募集を作成する手順

### 🛡️ 管理者向け — 設定

#### VC募集チャンネルをセットアップする

```
# カテゴリーを指定してセットアップする
/vc-recruit-config setup category:カテゴリー名

# カテゴリーなし（トップレベル）でセットアップする
/vc-recruit-config setup

# スレッドの自動アーカイブ時間を変更する（デフォルトは 24h）
/vc-recruit-config setup category:ゲームカテゴリー thread-archive:1h
```

実行すると、指定したカテゴリーに以下の2チャンネルが自動で作成されます。

| チャンネル名 | 役割                                                                                                                                          |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `#募集作成`  | 募集ボタンが設置されるチャンネル。メンバーはここのボタンから募集を作成する                                                                    |
| `#募集一覧`  | 投稿された募集メッセージが一覧表示されるチャンネル。直接書き込みは不可。各募集には Bot がスレッドを自動作成し、参加者はスレッド内で返信できる |

**オプション:**

| オプション       | 説明                                                                            |
| ---------------- | ------------------------------------------------------------------------------- |
| `category`       | 作成先カテゴリー名。省略するとコマンド実行チャンネルのカテゴリー（なければTOP） |
| `thread-archive` | 募集スレッドの自動アーカイブ時間。`1h` / `24h` / `3d` / `1w`（省略時は `24h`）  |

> 同一カテゴリーへの重複セットアップはできません。複数カテゴリーへのセットアップは可能です。

#### VC募集チャンネルを撤去する

```
/vc-recruit-config teardown
```

コマンドを実行すると、自分だけに見える操作パネルが表示され、以下の手順で削除を確定させます。

**ステップ 1 — 対象を選択する**

セットアップ済みのカテゴリーが選択肢に表示されます（複数選択可）。
撤去したいカテゴリーを選び、確定します。

**ステップ 2 — 確認する**

選択したカテゴリーの一覧が確認パネルに表示されます。

| ボタン      | 動作                           |
| ----------- | ------------------------------ |
| 🗑 撤去する | チャンネルセットを削除して完了 |
| 選び直す    | 対象選択に戻り、選択しなおせる |
| キャンセル  | 操作を中止する                 |

> ⚠️ 撤去すると募集作成・募集一覧チャンネルが削除されます。この操作は取り消せません。

#### メンション候補ロールを追加する

```
/vc-recruit-config add-role
```

コマンドを実行すると、自分だけに見える操作パネルが表示されます。
ロールセレクトメニューで追加したいロールを選択し（複数選択可）、「✅ 追加する」ボタンを押して確定します。
3分以内に操作してください（タイムアウトで自動キャンセル）。

追加したロールは、VC募集作成時のステップ1でメンション候補として表示されます。
最大25件まで追加できます。成功時は登録したロールが Embed で表示されます。

#### メンション候補ロールを削除する

```
/vc-recruit-config remove-role
```

コマンドを実行すると、登録済みのロール一覧がセレクトメニューで表示されます。
削除したいロールを選択し（複数選択可）、「🗑️ 削除する」ボタンを押して確定します。
3分以内に操作してください（タイムアウトで自動キャンセル）。
成功時は削除したロールが Embed で表示されます。

#### 現在の設定を確認する

```
/vc-recruit-config view
```

セットアップ済みのカテゴリーとチャンネル、メンション候補ロールの一覧が表示されます。

**表示例:**

<table border="1" cellpadding="8" width="380">
<tr><th align="left">🎤 VC募集設定</th></tr>
<tr><td>
<b>セットアップ済みカテゴリー</b><br>
• ゲームカテゴリー: #募集作成 / #募集一覧<br>
• TOP レベル: #募集作成 / #募集一覧
</td></tr>
<tr><td>
<b>メンション候補ロール</b><br>

</td></tr>

**⚠️ 注意事項:**

- セットアップには チャンネルの管理・閲覧権限が必要です。
- メンション候補にロールを追加しない場合、募集メッセージにメンションは付きません（「なし」のみ選択可）。
- 両チャンネルとも一般ユーザーの直接書き込みは不可です（インタラクション・スレッド返信は可）。

## ❓ よくある質問（FAQ）

### AFK機能について

**Q. AFK機能で「機能が無効です」と表示される。**
A. 管理者がまだ AFK チャンネルを設定していません。`/afk-config set-channel` を実行してください。

**Q. 他のメンバーをAFKに移動できない。**
A. 対象のメンバーがボイスチャンネルに参加していることを確認してください。VCに入っていないと移動できません。

---

### Bumpリマインダーについて

**Q. Bumpしたのにリマインダーが来ない。**
A. 以下を確認してください。

- ayasono がサーバーに参加しており、そのチャンネルを閲覧できる権限があるか
- `/bump-reminder-config view` で機能が有効になっているか
- メンション設定（ロール・ユーザー）が設定されているか、またはリマインダー登録ボタンを押しているか

---

### VC自動作成について

**Q. CreateVC に入ったのに VC が作られない。**
A. 以下を確認してください。

- ayasono がチャンネルを作成できる権限（`チャンネルの管理`）を持っているか
- `/vac-config view` でトリガーチャンネルが正しく設定されているか

**Q. VCのパネルが流れてしまって見つけられない。**
A. パネルは VC のチャット欄にあります。チャットを少しスクロールしてみてください。または `/vac vc-rename` などのコマンドを使う方法もあります。

---

### VC募集機能について

**Q. 「📢 VC募集を作成」ボタンが見当たらない。**
A. 管理者がまだセットアップを実施していません。`/vc-recruit-config setup` を実行してもらってください。

**Q. VC一覧に参加したいVCが表示されない。**
A. VC募集機能はセットアップしたカテゴリー内のVCのみを表示します。カテゴリーが別の場合は表示されません。

**Q. 「🎤 VCに参加」ボタンを押しても何も起きない。**
A. リンクボタンのため、クリックすると Discord 上でそのVCチャンネルへ移動し、参加ダイアログが表示されます。アプリ版とブラウザ版で挙動が異なる場合があります。

**Q. VCだけ終了して、投稿は記録として残しておきたい。**
A. 募集メッセージの「🔇 募集終了」ボタンを押してください。新規作成VCが削除され、投稿パネルが「🔇 募集終了済み」状態に更新されます。投稿・スレッドはそのまま残ります。投稿者本人または `MANAGE_MESSAGES` 権限を持つ管理者のみ操作できます。

**Q. 募集を取り消したい。**
A. 募集メッセージの「🗑️ 募集を削除」ボタンを押すと確認後に投稿・スレッド・新規作成VCをまとめて削除できます。投稿者本人または `MANAGE_MESSAGES` 権限を持つ管理者のみ操作できます。

**Q. 新規作成したVCの名前や人数制限を変えたい。**
A. 「🆕 新規VC作成」を選択して投稿すると、作成されたVCのテキストチャットに操作パネルが送られます。パネルの「✏️ VC名を変更」「👥 人数制限を変更」ボタンから設定できます。VC参加中のメンバーのみ操作できます。

**Q. VC募集チャンネルを撤去したい。**
A. `/vc-recruit-config teardown` を実行すると、セットアップ済みのカテゴリーの一覧から撤去対象を選択できます。確認パネルで「🗑 撤去する」を押すと募集作成・募集一覧チャンネルが削除されます。

**Q. VAC のトリガーチャンネルがVC一覧に表示されない。**
A. 仕様通りの動作です。VAC トリガーチャンネルは誤った参加を防ぐため、VC募集の選択肢から自動的に除外されます。

---

### メッセージ固定機能について

**Q. メッセージが最下部に表示されなくなった。**
A. Bot がチャンネルでメッセージを送信する権限があるか確認してください。また、`/sticky-message view` で設定が有効になっているか確認できます。

**Q. スティッキーメッセージを Embed で設定したい。**
A. `/sticky-message set channel:#チャンネル名 style:embed` を実行してください。ポップアップでタイトル・説明・カラーを入力できます。

## 🔧 Botに必要なサーバー権限

管理者の方は、ayasono に以下の権限が付与されていることをご確認ください。
権限が不足すると一部の機能が正常に動作しない場合があります。

| 権限                 | 必要な機能                   |
| -------------------- | ---------------------------- |
| メッセージを送信     | すべての機能                 |
| メッセージ履歴を読む | スティッキーメッセージ       |
| 埋め込みリンク       | Embed形式メッセージの表示    |
| メッセージの管理     | スティッキーメッセージの削除 |
| チャンネルの管理     | VC自動作成、VC募集           |
| メンバーを移動       | AFK機能、VC募集              |
